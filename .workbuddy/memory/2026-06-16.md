# 2026-06-16

## 修复 app.py 页面 404 问题

### 问题
项目能部署（Docker 启动成功），但访问 `/` 和 `/login` 等页面返回 404。
原因是 `app.py` 只包含了一个 `/ocr` POST 路由，缺少所有前端页面路由和业务 API。

### 解决
重写了完整的 `app.py`，包含：

1. **静态页面路由**：`GET /` → index.html, `GET /login` → login.html
2. **JWT 认证**：`/api/login` 登录/注册，`/api/whoami` 用户信息，`/api/logout` 登出
3. **报销 CRUD**：搜索分页、新增、编辑、删除
4. **统计数据**：按品类/人员汇总
5. **文本智能解析**：`/api/parse_text`
6. **Excel/PDF 导出**：`/api/export`
7. **图片文件服务**：`/api/image/<filename>`
8. **双模式 OCR**（保留原实现）：
   - 浏览器上传：multipart/form-data
   - 微信小程序：application/json + Base64
   - 改为异步任务模式（支持进度轮询）

### 关键变更
- 路由前缀从 `/ocr` 改为 `/api/ocr`，与前端一致
- OCR 改为异步任务模式：POST 返回 task_id，GET `/api/ocr/progress/<task_id>` 轮询进度
- 添加 `uploads/` 目录存储 OCR 截图
- 添加 JWT 认证装饰器 `@require_auth`
